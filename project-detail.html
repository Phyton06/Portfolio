TaskFlow

Plataforma de gestión de tareas en tiempo real con tableros colaborativos, automatización vía IA y pipelines de integración continua. Diseñada para equipos que necesitan moverse rápido sin perder el control.

Repo phyton/taskflow
Rol Full-stack Lead
Estrellas 0
Commits 0
Duración 6 meses

Arquitectura

TaskFlow está construido con una arquitectura hexagonal donde el dominio es el centro de todo. Cada capa tiene una responsabilidad clara y se comunica a través de puertos y adaptadores. El backend en Python con FastAPI maneja la lógica de negocio, mientras que el frontend en React consume una API REST tipada con Pydantic.

# app/domain/entities/task.py from dataclasses import dataclass from enum import Enum class TaskStatus(Enum): TODO = "todo" IN_PROGRESS = "in_progress" REVIEW = "review" DONE = "done" @dataclass class Task: id: str title: str status: TaskStatus assignee: str | None

APIs y Contratos

Toda la comunicación entre frontend y backend está tipada. Los contratos se definen con Pydantic en el backend y se reflejan 1:1 en el frontend con TypeScript. Esto elimina bugs de interfaz y hace que el código sea auto-documentado.

# app/api/schemas/task.py from pydantic import BaseModel class TaskCreate(BaseModel): title: str description: str | None = None class TaskResponse(BaseModel): id: str title: str status: TaskStatus created_at: datetime

Despliegue y DevOps

Cada push a main gatilla un pipeline de CI/CD que corre tests, linters, builds la imagen Docker y deploya a un clúster ECS en AWS. El pipeline completo tarda ~4 minutos y cubre seguridad, calidad de código y tests de integración.

# .github/workflows/deploy.yml name: deploy on: push: branches: [main] jobs: deploy: runs-on: ubuntu-latest steps: - uses: actions/checkout@v4 - run: docker build -t taskflow . - run: docker push ${{ secrets.ECR_REPO }} - run: aws ecs update-service ...
